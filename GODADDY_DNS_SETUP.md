   ```bash
   # Generate private key
   openssl genrsa -out dkim_private.key 1024

   openssl rsa -in dkim_private.key -pubout -out dkim_public.key

   cat dkim_public.key | grep -v "^-" | tr -d '\n'
   ```

DNS changes can take up to 48 hours to propagate globally. Check status:

### Using Online Tools:
1. **MXToolbox:** https://mxtoolbox.com/SuperTool.aspx
   - Enter: `htresearchlab.com`
   - Check MX, A, SPF, DKIM, DMARC records

2. **DNSChecker:** https://dnschecker.org
   - Enter: `htresearchlab.com`
   - Select record type: MX, A, TXT
   - Check propagation across different locations

### Using Command Line:

```bash
# Check MX records
nslookup -type=MX htresearchlab.com

# Check A record for mail subdomain
nslookup mail.htresearchlab.com

# Check SPF record
nslookup -type=TXT htresearchlab.com

# Check DKIM record
nslookup -type=TXT default._domainkey.htresearchlab.com

# Check DMARC record
nslookup -type=TXT _dmarc.htresearchlab.com
```

After DNS is configured, update your email server settings:

# Update these values

## Troubleshooting

### Problem: MX record not resolving
**Solution:**
- Wait 2-4 hours for DNS propagation
- Check if TTL expired and record is cached
- Verify MX record priority is set to 10

### Problem: Emails going to spam
**Solutions:**
- Verify SPF, DKIM, DMARC records are correct
- Set up PTR (reverse DNS) with hosting provider
- Check spam score at https://www.mail-tester.com
- Warm up your IP (send gradually increasing volumes)

### Problem: Cannot receive emails
**Solutions:**
- Verify SMTP server is running on port 2525
- Check firewall allows port 2525 inbound
- Confirm A record points to correct IP
- Test with: `telnet mail.htresearchlab.com 2525`

### Problem: DKIM validation failing
**Solutions:**
- Verify public key matches private key
- Remove line breaks from DKIM TXT record
- Check selector is "default" in both DNS and server config
- Wait for DNS propagation

## Security Recommendations

1. **Enable DNSSEC** (in GoDaddy DNS settings)
   - Protects against DNS spoofing attacks

2. **Set up SSL/TLS Certificate** (Let's Encrypt)
   ```bash
   sudo certbot --nginx -d mail.htresearchlab.com
   ```

3. **Change SMTP Port to 465/587 for production**
   - Port 2525 is for testing
   - Port 587 (STARTTLS) or 465 (SSL/TLS) for production

4. **Enable Firewall Rules**
   ```bash
   # Allow SMTP ports
   sudo ufw allow 587/tcp
   sudo ufw allow 465/tcp
   sudo ufw allow 2525/tcp
   ```

5. **Monitor Email Logs**
   - Review server logs regularly
   - Set up DMARC reports monitoring
